## What Was Fixed This Session

1. **memfs git blockage** — `system/human.md` and `system/persona.md` were
   physically deleted (memory restructure) but never committed. Git refused
   to pull. Fixed by staging and committing the deletions.

2. **501 on new-agent clone** — `LETTA_MEMFS_LOCAL` and `LETTA_MEMFS_SERVICE_URL`
   were only in `.bashrc`, not as Windows user env vars. Server launched outside
   Git Bash didn't see them. Fixed by setting both as permanent user env vars
   and verifying `MemfsClient initialized` in server startup output.

3. **Push rejected on non-bare repos** — All real agent repos under
   `~/.letta/memfs/repository/` were initialized as `bare = false` (Letta
   server created them directly). Pushes failed with "refusing to update
   checked out branch". Fixed by adding `receive.denyCurrentBranch = ignore`
   to each repo config. Script: `scripts/fix-repo-configs.ps1`.

4. **Three ENAMETOOLONG patches to letta-code v0.19.6** — Windows command
   line length limits (8,191 for cmd.exe, 32,767 for CreateProcess) were
   exceeded in three places. Script to reapply: `scripts/apply-patches.ps1`

## Restore Procedure (If Something Breaks)

```powershell
# 1. Restore env vars
.\scripts\restore-env.ps1

# 2. Reapply letta.js patches (after any npm update)
.\scripts\apply-patches.ps1

# 3. Fix any new agent repo configs (after push failures)
.\scripts\fix-repo-configs.ps1
```

Then restart everything in startup sequence order.
